Introduction to Python

The Python programming language was created in 1991 by the Dutchman Guido van Rossum.


His name - Python (or Python) - received from the name of the television series, and not the reptile.


After Rossum developed the language, he released it to the Internet, where the programming community joined in to improve it.


Python is actively developing now. New versions are released frequently. Until recently, two separate branches of the language were supported: Python 2.x and Python 3.x. Here, the English letter "x" denotes a specific release. There is a slight difference between the second and third Python. Support for Python 2 is currently deprecated.


Python programming language

Python is a high-level general-purpose programming language that is also used to develop web applications. The language is focused on improving developer productivity and code readability.

The correct Russian pronunciation of the name of the programming language is Python, but the distorted one is more often used - Python.

Python supports several programming paradigms: structural, object-oriented, functional, imperative, and aspect-oriented. The language has dynamic typing, automatic memory management, full introspection, an exception handling mechanism, support for multi-threaded computations, and convenient high-level data structures. Python code is organized into functions and classes, which can be combined into modules, which in turn can be combined into packages. Python is usually used as interpreted, but can be compiled to Java bytecode and to MSIL (within the .NET platform).

The developers of the Python language adhere to a certain programming philosophy called "The Zen of Python" ("Zen of Python" or "Zen of Python"):

  • Beautiful is better than ugly.
  • Explicit is better than implicit.
  • Simple is better than complex.
  • Complicated is better than confusing.
  • Flat is better than nested.

    • Compared to Ruby and some other languages, Python lacks the ability to modify built-in classes such as int, str, float, list, and others.

    Python has a global interpreter lock (GIL) - during its work, the main interpreter constantly uses a large amount of thread-unsafe data. Basically, these are dictionaries that store the attributes of objects, and access to external code, therefore, in order to avoid the destruction of this data during joint modification from different threads, before the execution of several instructions (usually a hundred), the interpreter thread captures the GIL, and releases it at the end.

    There are several implementations of the language - CPython (main), Jython, PyS60, IronPython, Stackless, Python for .NET, PyPy, python-safethread, Unladen Swallow, tinypy.
